## Intelligent Features

### Content Pruning
**LLM-based relevance extraction**:

1. **Token Estimation**: Estimate content size
2. **Relevance Analysis**: Use LLM to identify relevant sections
3. **Smart Extraction**: Extract most relevant parts
4. **Structure Preservation**: Maintain code structure

**Configuration**:
- Threshold: 1250 tokens (~5000 characters)
- Controlled by `explanation` parameter quality
- Can be disabled with `skipPruning=true`

### AST Analysis
**Code-aware operations**:
- Parse code into Abstract Syntax Tree
- Extract function/class signatures
- Understand code structure
- Safer refactoring

**Tools using AST**:
- `readCode`: AST-based code reading
- `editCode`: AST-based code editing
- `rename_symbol`: Semantic renaming
- `extract_method`: Code extraction

### Token Management
**Context window optimization**:
- Token estimation for all content
- Intelligent pruning when needed
- Budget tracking
- Context window checking

### Fallback Mechanisms
**Error recovery strategies**:
- `undo_last_edit`: Restore previous version
- Retry with alternative approaches
- Graceful degradation
- Clear error messages

## Hook System

### Event-Driven Automation
**Hook Types**:
- `preToolUse`: Before tool execution
- `postToolUse`: After tool execution
- `fileEdited/Created/Deleted`: File system events
- `promptSubmit/agentStop`: Conversation events
- `preTaskExecution/postTaskExecution`: Task lifecycle

### Hook Actions
- `askAgent`: Send message to agent
- `runCommand`: Execute shell command

### Use Cases
- Access control (preToolUse)
- Logging (postToolUse)
- Auto-linting (fileEdited)
- Auto-testing (postTaskExecution)

## Sub-Agent Architecture

### Specialized Agents
**Delegation pattern**:
- `context-gatherer`: Repository exploration
- `general-task-execution`: General tasks
- `custom-agent-creator`: Create new agents

### Benefits
- Specialized expertise
- Isolated context
- Parallel execution
- Focused problem-solving

### When to Use
- Unfamiliar codebase
- Complex multi-step tasks
- Specialized domains
- Parallel work streams

## Response Style

### Personality Traits
1. **Knowledgeable, not instructive**: Show expertise without condescension
2. **Decisive and clear**: Be precise, lose the fluff
3. **Supportive, not authoritative**: Compassionate and welcoming
4. **Easygoing, not mellow**: Calm and relaxed, but engaged
5. **Speak like a dev**: Technical when needed, digestible otherwise

### Communication Principles
- Be concise and direct
- Don't repeat yourself
- Prioritize actionable information
- Minimal summaries (no bullet lists)
- Don't create unnecessary documentation

## Key Insights

### Insight 1: State Machine > Loop
LangGraph provides:
- Structured execution
- Conditional branching
- State preservation
- Error recovery

### Insight 2: Claude Decides, Metis runtime Executes
- Claude chooses tools
- Claude decides parallel/sequential
- Metis runtime executes efficiently
- Clear separation of concerns

### Insight 3: Intelligent Pruning is Critical
- Not blind deletion
- LLM-based relevance
- Saves tokens
- Maintains quality

### Insight 4: Hooks are Powerful
- Event-driven automation
- Access control
- Workflow customization
- Infinite possibilities

### Insight 5: Sub-Agents Enable Specialization
- Focused expertise
- Isolated context
- Parallel execution
- Better results
