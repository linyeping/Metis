## Parameter Discipline

### Required vs Optional
**Strict distinction**:
- Required parameters: Must be provided
- Optional parameters: Have sensible defaults
- Never make up values for optional parameters
- Never ask about optional parameters unless necessary

### Exact Values
**User intent respected**:
- If user provides value in quotes, use EXACTLY
- Don't modify, interpret, or "improve"
- Respect explicit intent
- Maintain user control

### Parameter Validation
**Before execution**:
- Check all required parameters provided
- Validate parameter types
- Check parameter values
- Clear error messages if invalid

## C-Style Aliases

### Short Names
**Convenience aliases**:
- Read → read_file
- Write → write_file
- Shell → execute_bash_command
- Grep → grep_search
- Glob → glob_search
- StrReplace → robust_replace_in_file
- Delete → delete_file

### Benefits
- Faster to type
- Easier to remember
- Consistent with Metis tool discipline
- Backward compatible

### Implementation
```python
TOOL_NAME_ALIASES = {
    "Read": "read_file",
    "Write": "write_file",
    "Shell": "execute_bash_command",
    # ... more aliases
}

def canonical_tool_name(name):
    return TOOL_NAME_ALIASES.get(name, name)
```

## Parameter Mapping

### Automatic Normalization
**C-style to implementation**:
- `path` ↔ `file_path`
- `contents` ↔ `content`
- `old_string` ↔ `search_text`
- `new_string` ↔ `replace_text`
- `glob_pattern` ↔ `pattern`
- `target_directory` ↔ `root`
- `search_term` ↔ `query`
- `description` ↔ `prompt`
- `working_directory` ↔ `cwd`
- `offset` + `limit` → `start_line` + `end_line`
- `block_until_ms` → `timeout` (ms to seconds)

### Benefits
- Supports both naming conventions
- Smooth migration Metis tool discipline
- Maintains compatibility
- User choice

## Structured Workflows

### Tool Collaboration
**Designed to work together**:

1. **Read → Analyze → Modify**
   ```
   Read file → Understand → StrReplace
   ```

2. **Search → Read → Fix**
   ```
   Grep → Read → StrReplace → Shell (test)
   ```

3. **Parallel Modifications**
   ```
   StrReplace(file1) + StrReplace(file2) + StrReplace(file3)
   All execute simultaneously
   ```

### Clear Patterns
- Sequential when dependent
- Parallel when independent
- Verify after modifications
- Test after changes

## Execution Patterns

### Parallel Execution
**Key insight Metis tool discipline**:
- When tools are independent, execute in parallel
- Dramatically improves performance
- Claude decides, system executes
- Use Promise.all or equivalent

**Example**:
```python
# All execute simultaneously
robust_replace_in_file(file_path="a.py", ...)
robust_replace_in_file(file_path="b.py", ...)
robust_replace_in_file(file_path="c.py", ...)
```

### Sequential Execution
**When needed**:
- Tool B depends on Tool A's result
- Order matters
- State changes between calls

**Example**:
```python
# Sequential
content = read_file(file_path="config.py", ...)
robust_replace_in_file(file_path="config.py", search_text=content[10:20], ...)
```

## Error Handling

### Clear Error Messages
**Actionable feedback**:
- What went wrong
- Why it went wrong
- How to fix it
- Alternative approaches

**Example**:
```
❌ 参数错误 (robust_replace_in_file): oldStr not found in file
传入键: ['file_path', 'search_text', 'replace_text']

Suggestion: Check that search_text exactly matches file content (including whitespace)
```

### Error Recovery
- Don't retry exact same call
- Try alternative approaches
- Explain to user
- Suggest fixes

## Security Baseline

### HTTPS Only
**Web operations**:
- All web_fetch requires HTTPS
- No HTTP allowed
- No exceptions
- Security over convenience

### SSRF Protection
**Server-Side Request Forgery**:
- No localhost URLs
- No internal IP ranges
- DNS resolution checked
- Prevents internal network access

### Path Validation
**File operations**:
- Workspace boundary enforced
- No path traversal
- Symlink validation
- Clear error messages

## Tool-Specific Features

### StrReplace
**Precise replacement**:
- `replace_all` parameter for multiple occurrences
- Exact matching required
- Whitespace-sensitive
- Include context for uniqueness

### EditNotebook
**Multiple modes**:
- `is_new_cell`: Insert new cell
- `old_string`/`new_string`: Block replacement
- `new_content`: Whole cell replacement

### Grep
**Advanced search**:
- `output_mode`: matches/files/count
- `head_limit`: Limit results
- Regex support
- Context lines

### AskQuestion
**Structured questions**:
- Schema version 2
- Multiple choice
- Free-form text
- Structured responses
